- 클로저는 외부함수 안에 있는 내부함수에서 외부함수의 변수를 참조할때, 내부함수에서도 외부함수의 변수를 참조할수 있다는 것입니다.
- 클로저의 특징은 외부함수가 종료가 되었어도  외부함수의 변수를 내부함수에서 참조가 가능하다는 것입니다. 

그렇다면 클로저는 어디에 쓰이나요? 써본적이 있나요? 

클로저는 상태를 안전하게 저장하고 변경하는데 쓰입니다.

- 제가 자바스크립트로 리액트의 useState 를 구현해볼때 써보았습니다. 

  ```js
  function useState(initialValue) {
    var _val = initialValue // _val은 useState에 의해 만들어진 지역 변수입니다.
    function state() {
      // state는 내부 함수이자 클로저입니다.
      return _val // state()는 부모 함수에 정의된 _val을 참조합니다.
    }
    function setState(newVal) {
      // 마찬가지
      _val = newVal // _val를 노출하지 않고 _val를 변경합니다.
    }
    return [state, setState] // 외부에서 사용하기 위해 함수들을 노출
  }
  ```

  - 함수형 프로그래밍을 위함 pipe 함수를 구현할때 또한 클로저를 이용해서 pipe 함수를 만들었습니다.

`HttpRequest`와 [Fetch API](https://developer.mozilla.org/ko/docs/Web/API/Fetch_API)는 **동일 출처 정책**(same-origin policy)을 따릅니다.

동일출처 정책에서 동일 출처라는 것은 프로토콜과 호스트 가 모두 같아야 동일한 출처라고 말합니다.  

어떤 [출처](https://developer.mozilla.org/ko/docs/Glossary/Origin)에서 불러온 문서나 스크립트가 다른 출처에서 가져온 리소스와 상호작용하는 것을 제한하는 중요한 보안 방식

>  콜백함수는 무엇인가요?

함수의 매개변수를 통해 함수의 외부에서 내부로 전달받은 함수를 콜백함수 라고합니다. 



> 호이스팅은 무엇인가요?

함수를 선언문으로 선언하면 함수 선언문이 코드 맨위로 끌어올려진 것처럼 동작하는 것을 말합니다. 



힘스 선언문으로 생성한 함수와 표현문으로 생성한 함수의 생성 시점이 다르기 때문이다. 함수 선언식을 자바스트립트 런타임 이전에 함수객체가 먼저 생성이 된다. 그렇기 때문에 함수 이름과 같은 식별자를 암묵적으로 생성하고  함수 표현식은  자바스크립트 런타임 내에서 함수를 변수에 할당하는 코드를 읽을 때 식별자를 생성한다. 



> 실행 컨텍스트 가 몬가요

소스코드의 평가하면서 실행하는데 필요한 환경(스코프)를 생성하고  실행을 관리하면서 실행 결과들을 스코프에 저장하는역할을 합니다.


식별자와 스코프는 렉시컬 환경을 관리하고 코드의 실행순서는 실행 컨텍스트 스택으로 관리합니다. ( 콜스택 )
